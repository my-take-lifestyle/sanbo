PART 1

はじめに — このアプリは何をする道具か

「参謀」は、テーマ投資をする個人のための意思決定支援アプリです。証券口座には接続せず、売買もしません。代わりに「資産の現状を見える化する」「情報を自分のテーマに紐付けて蓄積する」「見直すべき条件（トリガー）を管理する」「判断を記録して振り返る」の4つを引き受けます。

やることやらないこと
資産の現状を常に見える化する証券口座接続・自動売買・自動発注
あなたの専門知識を投資仮説に変換する導線を作る銘柄の断定的な売買推奨
「いつ・何を見直すか」の条件（トリガー）を管理する網羅的な家計簿（明細管理）
判断の記録と振り返り（ジャーナル）完全な取引履歴の複製（証券会社に既にある）

3つの設計思想

  • 週15分で回る。入力コストが週15分を超える機能は載せていません。毎日開く必要はありません。
  • 手動入力がプライマリ、API は加速装置。為替・米国株価は自動取得できますが、取得に失敗しても前回値のまま全機能が動き続けます。API キーがなくても困りません。
  • データは端末の中だけ。資産データはブラウザの localStorage にのみ保存され、外部送信されません（API へ送るのはティッカー記号だけ）。裏返すとブラウザのサイトデータを消すと全部消えるので、バックアップが生命線です（→ 3-7）。

PART 2

画面ガイド（ビジュアル）

各画面のどこに何があるかを、実際の表示そのままの言葉で示します。図はサンプルデータ読み込み時の表示例です。

🏠 司令部（ダッシュボード）

  1. 1総資産と前月比。前月スナップショットとの差を、為替影響とそれ以外にざっくり分けて表示します。
  2. 2為替と投資余力。USD/JPY はいつ・どこから取得した値かを常に明示。月次投資可能余力（概算） = 収入 − 生活費 − ローン − 積立（設定のキャッシュフローから計算）。
  3. 3待機資金メーター。現金 + MMF の比率。白い縦線が目標上限（初期値 25%、設定で変更可）。超過すると金額と投入候補テーマが示されます。
  4. 4配分。クラス別 / テーマ別を切替。テーマ別では「仮説あり・エクスポージャーゼロ」のテーマも気づきとして出ます。
  5. 5要アクション。トリガー成立・判断レビュー期限・バックアップ警告・未トリアージ件数・放置検知などが自動で集まります。タップで該当画面へ。
  6. 6確認済みボタン。成立したトリガーはこれで消し込み。条件が一度外れて再び成立すると再表示されます。
  7. 7📈 モデル / ⚙️ 設定への入口。

📊 資産（ポートフォリオ）

  1. 1為替を更新 — Frankfurter（無料・キー不要）から USD/JPY を取得。失敗しても前回値のまま動きます。
  2. 2米国株価を一括更新 — Finnhub キー設定時のみ有効。保有・ウォッチ銘柄・モデルのピックの価格と SPY をまとめて更新し、価格トリガーも自動判定します。
  3. 3口座管理 — 銀行・証券などの口座の追加・編集。保有は必ずどれかの口座に紐付きます。
  4. 4価格の鮮度表示。各行に「いつ・どの経路の価格か」（manual / finnhub）を表示。90日超は黄色で警告。行タップで編集画面が開きます。
  5. 5＋ 保有を追加 — 現金は「数量 = 金額、価格 = 1」で登録。投信は「数量 = 万口、価格 = 1万口あたり基準価額」のような単位でも OK（評価額 = 数量 × 価格 × 為替、で計算されるため）。

🧭 テーマ & ウォッチ

  1. 1テーマカード。確信度（1〜5）・円換算エクスポージャー・紐付いたインテル数・KPI を一望。タップで詳細（仮説（thesis）・自分の情報優位（myEdge）・保有・ウォッチ・インテル・タイムライン）が開き、編集 / 📋 テーマレビューボタンがあります。
  2. 2myEdge 警告。「なぜ自分が市場平均より良い判断ができるか」を書けないテーマには警告が出ます。このアプリの一番の思想です。
  3. 3⏰ トリガー成立 — 価格条件が成立中の銘柄に付くバッジ。
  4. 4トリガー（見直し条件）。種類は株価が下回ったら / 株価が上回ったら / イベントの3つ。価格系は自動判定、イベント系は手動確認です。
  5. 5参考シグナル。この銘柄を現行ピックに含むモデル名。タップでスコアボードへ。一視点であり推奨ではありません。

📥 インテル受信箱

  1. 1AI 連携の中枢。📋 コンテキストパックを生成してコピー→ AI チャットに貼る → 回答全文を貼り戻し欄へ →回答を取り込む。週次ブリーフィング・テーマレビュー・モデル実行、どの回答もここで受けられます。
  2. 2クイック投入。URL は「しおり」として投げ込むだけ（本文の要約は AI 連携で）。展示会で聞いた話などの一次情報は業務知見（work_insight）にチェックしてメモ。
  3. 3自動収集。GitHub Actions が毎朝拾った公開ニュース。取り込むを押すまでアプリのデータには入りません。
  4. 4未トリアージ = 受信箱。テーマ未紐付けのものが上に積まれます。カードをタップ →テーマ（紐付けるとトリアージ完了）にチェックして保存。空にすると「受信箱は空です（Inbox Zero 🎉）」。

📓 ジャーナル

  1. 1＋ 起票 — 種別は買い / 売り / 一部売却 / 買い増し / 見送り / リバランス。金額は概算で OK。判断理由（当時の仮説を残す）が本体で、根拠にしたインテルも紐付けられます。
  2. 2レビュー期限は起票時に +6か月で自動設定（編集可）。
  3. 3期限が来ると司令部の要アクションに「判断レビュー期限」が出ます。開いて判断の質（良い判断/妥当/悪い判断）と結果（良い結果/中立/悪い結果）を別々に記録し、学び（lesson）を残すと完了です。

📈 モデル・スコアボード

  1. 1モデルとは「選定ロジックの言語化（logicPrompt）+ 仮想ポートフォリオ + 成績記録」。実際の売買は一切しません。初期3本（テーマ・モメンタム型 / コントラリアン型 / クオリティ・バリュエーション型）が入っています。カードをタップすると詳細（出典・由来（origin）・選定ロジック（logicPrompt）・ピック履歴・retire）。
  2. 2成績。累積相対リターン = ベンチマーク（米国株→S&P500（SPY）、日本株→TOPIX（1306））に対する超過分の weight 加重平均。ヒット率 = ベンチマークに勝ったピックの割合。
  3. 3月次実行。各モデルの 📋 ボタンで実行プロンプトをコピー → AI の回答を貼り戻し →ピックを取り込む。仮想エントリー価格は自動設定されます。
  4. 4ベンチマーク。SPY は米国株価を一括更新で自動記録。1306 は今日の値として記録で手動入力。記録が無い期間は「欠損」として計算からスキップされます（壊れません）。

PART 3

取扱説明書（操作手順）

3-1. 最初の30分（セットアップ）

  1. まずサンプルで触る。⚙️ 設定 →データ→サンプルデータを読み込む。全画面がどう見えるか一巡してから、全データ消去で白紙に戻します。
  2. スマホにインストール。Android Chrome でアプリの URL を開き「ホーム画面に追加」。これで共有メニューからのインテル投入（Share Target）が使えるようになります。
  3. 設定を埋める。⚙️ 設定で目標・キャッシュフロー（月次の代表値）・固定プロフィールを入力。制限リスト（取引自粛銘柄）には自社・協業先などを必ず登録してください。
  4. （任意）Finnhub キー。finnhub.io で無料登録し、キーをAPI・為替に保存すると米国株価の一括更新が使えます。無くても全機能は手動で動きます。
  5. 口座と保有を登録。📊 資産 →口座管理で口座を作り、＋ 保有を追加で保有を登録。細かい正確さより「全体が見える」ことを優先してください。
  6. テーマを1つ書く。🧭 テーマ →＋ 追加。仮説（thesis）と自分の情報優位（myEdge）は必ず言葉にします。
  7. 最初のバックアップ。⚙️ 設定 →JSON エクスポート。ここが30日警告の起点になります。

3-2. 資産の登録と価格更新

  • 評価額は入力しない。常に「数量 × 価格 × 為替」で自動計算されます。買い増したら数量と取得単価（現地通貨）を書き換えるだけ。取引履歴は持ちません（証券会社にあるので）。
  • 手動価格の更新は行をタップ →価格（1単位）を書き換えて保存。source が manual、基準日が今日になります。
  • 定額積立（新NISA・金銀積立など）は保有の定額積立（円/月）に金額を入れると、投資可能余力の計算に反映されます。
  • 月次スナップショットは、月が変わって最初にアプリを開いたとき自動記録されます。手動でやり直すなら ⚙️ 設定 →当月スナップショットを再記録。

3-3. インテルの投入とトリアージ

  • 入口は3つ。①スマホの共有メニューから「参謀」を選ぶ（最速） ②📥 インテルのURL 追加 / メモ追加 ③自動収集から取り込む。
  • トリアージ = テーマに紐付けること。未トリアージのカードをタップし、テーマにチェック・センチメント（好材料/懸念/中立）・so-what（自分の PF・テーマへの示唆）を書いて保存。対応が必要なら要対応（ダッシュボードに表示）にチェック。
  • 読み終わったらアーカイブ。削除せず残すことでテーマのタイムラインに蓄積されます。

3-4. AI 連携（コピペ方式）— このアプリの核

アプリはあなたのテーマ・配分（比率）・ウォッチ・未処理インテルを1つのプロンプト（コンテキストパック）に自動合成します。毎回ゼロからプロンプトを書く必要はありません。

  1. 📥 インテル →📋 コンテキストパックを生成してコピー
  2. Claude などの AI チャットに貼り付けて送信
  3. 回答の全文をコピーし、貼り戻し欄にペースト →回答を取り込む
  4. 回答末尾の JSON が自動で展開されます — 要約インテル（テーマ・銘柄に自動紐付け）、テーマごとの評価（レビューノートに追記）、見落とし指摘（要対応インテルとして登録）
失敗しても壊れません。JSON が読めない回答は「全文を1枚のインテルとして保存します」。また、AI に渡す資産情報は初期設定では比率のみです（⚙️ 設定 →AI 連携で変更可）。

同じ貼り戻し欄で受けられる回答は3種類あります: 週次ブリーフィング（上記）、テーマレビュー（テーマ詳細の📋 テーマレビューから生成）、モデル実行（📈 の月次モデル実行（コピペ）から生成。こちらは 📈 画面の貼り戻し欄でも可）。

3-5. トリガーとジャーナル

  • トリガーは「事前に決めた見直し条件」。ウォッチ銘柄の編集画面で＋ 条件を追加。価格系は価格更新のたびに自動判定され、成立すると司令部に「トリガー成立」が出ます。確認済みで消し、条件が再成立すれば再表示。
  • 動いたら（見送っても）起票。📓 ジャーナル →＋ 起票。売買した時だけでなく見送りも記録します — 見送り判断の振り返りは買い判断と同じくらい学習価値が高いからです。
  • 6か月後、司令部に「判断レビュー期限」が出たら開いて、判断の質と結果を分けて記録。「良い判断だったが結果は悪かった」を区別できることが、学習の質を決めます。

3-6. モデル選定エンジン（月次）

  1. 月初、司令部に「モデル実行（月次）: 今月のピック未記録」が出ます → 📈 を開く
  2. 各モデルの 📋 ボタンで実行プロンプトをコピー → AI に貼る
  3. 回答を貼り戻してピックを取り込む。エントリー価格は保有/ウォッチの既知価格や Finnhub から自動設定（日本株は手動）
  4. あとは米国株価を一括更新のたびに仮想成績が自動追跡されます
  5. 四半期ごとにスコアボードを見て、成績下位モデルをretire（履歴は消えません）、新しい視点のモデルを＋ モデル追加
ピックは参考シグナルであり推奨ではありません。目的は「複数の直交する視点に仮説を出させ、実績で選別する」実験です。上位モデルのピックはウォッチリストに 📈 バッジで表示されますが、あくまで一視点です。

3-7. データ保全（最重要）

データはこの端末のブラウザ内にのみ保存されます。ブラウザの「サイトデータを削除」で全消失します。30日以上バックアップが無いと司令部に警告が出ます。
  • 基本: ⚙️ 設定 →JSON エクスポート。sanbo-backup-日付.json がダウンロードされます。
  • PC の Chrome/Edge なら: 自動バックアップ（ワンタップ上書き）で一度保存先を指定すれば、以後は今すぐ上書きバックアップの1タップで同じファイルに上書きできます。
  • 復元・機種変更: 新しい端末でアプリを開き、⚙️ 設定 →JSON インポートでバックアップファイルを選ぶだけ。完全復元されます（インポート前の状態も自動退避されます）。
PART 4

おすすめの使い方（運用リズム）

このアプリは「毎日開くアプリ」ではなく「週1の儀式 + 随時インテル投入」に最適化されています。本業が多忙でも破綻しないリズムがこれです。

随時 — 数秒
出会った情報は、その場で投げ込むだけ
  1. 記事を見つけたら共有メニュー →「参謀」（または URL をコピーしてURL 追加）
  2. 仕事で聞いた一次情報はメモ追加 +業務知見（work_insight）にチェック
ここで読み込まない・整理しないのがコツ。判断は週末にまとめて。あなたの情報優位は「仕事で得た高品質な情報」にあり、それを漏らさず溜める導線がこの数秒です。
週次 — 15分（週末推奨）
週次ブリーフィング — このアプリの心臓部
  1. 3分: 📊 資産 →為替を更新→米国株価を一括更新。気になる手動銘柄だけ価格を書き換え
  2. 5分: 📥 インテルの未トリアージを捌く（テーマ紐付け + so-what 一言）。自動収集も一瞥して気になるものだけ取り込む
  3. 5分: 📋 コンテキストパックを生成してコピー→ AI →回答を取り込む
  4. 2分: 🏠 司令部で要アクションを確認。トリガー成立を確認済みにし、動く/見送るを決めたら 📓 で＋ 起票
イベント型トリガー（決算開示待ちなど）はこのタイミングで目視確認します。15分で終わらない週は、入力を増やしすぎのサイン。
月次 — 10分（月初）
月替わりの儀式
  1. 月が変わって最初に開いた時、当月スナップショットが自動記録されます（操作不要）
  2. 日本株・投信・金銀など手動銘柄の価格を月1で更新（このあと当月スナップショットを再記録を押すと反映済みの値で取り直せます）
  3. 📈 でモデル実行: 📋 でプロンプトコピー → AI →ピックを取り込む。TOPIX（1306）の値も今日の値として記録
スナップショットが貯まると、待機資金比率と配分の推移が時系列で追えるようになります。「寝かせている資金」の改善が数字で見える仕組みです。
四半期 — 30分
仮説とモデルの棚卸し
  1. 各テーマの詳細を開き📋 テーマレビュー→ AI →貼り戻し。反証材料・KPI 状況・エクスポージャー過不足がレビューノートに蓄積されます
  2. 司令部の放置検知（インテルが3か月途絶えたテーマ）を確認 — テーマ投資の失敗は「買った後の追跡停止」で起きます。追う気が失せたテーマは状態を終了に
  3. 📈 スコアボードで成績比較 → 下位モデルをretire、新モデル投入（出典・由来（origin）に着想元をメモ）
  4. 📓 のレビュー待ちを消化
仮説に都合の良い情報だけ集めるのを防ぐため、テーマレビューのプロンプトは反証材料を最優先で聞く作りになっています。

PART 5

困ったとき（FAQ）

為替や株価が取得できない / オフラインで開いた
そのまま使えます。アプリは常に「前回値 + いつの値か（asOf）」で動き続ける設計です。為替は ⚙️ 設定 →API・為替の手動設定でいつでも上書きできます。オフラインでも全画面が開きます（PWA キャッシュ）。
保有の価格が黄色く表示される
その価格の基準日（asOf）が90日を超えています。鮮度の見える化であって異常ではありません。行をタップして価格（1単位）と価格の基準日を更新してください。
AI の回答を貼り戻したら「JSON を解釈できなかったため、全文を1枚のインテルとして保存しました」と出た
回答末尾に規定の JSON ブロックが無かったケースです。データは失われていません（全文が受信箱に入っています）。AI チャット側で「先ほど指定した JSON フォーマットで出力し直して」と頼み、その回答を貼り直せば通常どおり展開されます。コンテキストパックを使わず自由に質問した回答は、この生テキスト保存になるのが正常動作です。
モデルの回答を貼り戻したら「モデル名を解決できませんでした」と出た
回答 JSON 内の model 名がスコアボードのモデル名と一致していません。実行プロンプトには「model 名は変えずにそのまま返す」指示が入っていますが、AI が変えてしまうことがあります。回答の JSON 内のモデル名をスコアボードの表記に手で直して貼り直してください。
スコアボードの相対リターンが「—」や「欠損」になる
ピック日周辺のベンチマーク価格（S&P500（SPY）または TOPIX（1306））が記録されていないためです。SPY は米国株価を一括更新で自動記録されます。1306 は 📈 画面で今日の値として記録から手動入力してください。欠損期間のピックは計算から除外されるだけで、他は正常に集計されます。
スマホの共有メニューに「参謀」が出てこない
共有ターゲットは PWA としてインストールした場合のみ有効です。Android Chrome でアプリを開き「ホーム画面に追加」してください。なお iOS の Safari はこの機能（Share Target）に対応していないため、iPhone では URL コピー →URL 追加を使ってください。
司令部から「トリガー成立」を消したのにまた出てきた
仕様です。確認済みは「今回の成立を確認した」印で、価格が一度条件圏外に出て再び条件を満たすと、新しい成立として再表示されます。恒久的に止めたい場合はウォッチ銘柄の編集画面でそのトリガー行を × で削除してください。
自動収集に「新着はありません」と出続ける
2つの可能性があります。① GitHub Actions がまだ動いていない — リポジトリの Actions タブで「Update feed.json」を Run workflow して確認。② キーワードに合う記事が無かった — config/feed-keywords.json のキーワードを実際のテーマ・銘柄に合わせて増やしてください（空配列にすると全件通過）。
機種変更・別端末への引っ越し
旧端末: ⚙️ 設定 →JSON エクスポート。ファイルを新端末に渡し、新端末でアプリを開いてJSON インポート。これで完全復元です。端末間の自動同期機能はありません（データを外部に出さない設計のため）。
間違えてインポート/サンプル読み込みをしてしまった
直前の状態はブラウザ内に自動退避されています（キー: sanbo.appState.preImportBackup）。開発者ツールの localStorage からこの値を sanbo.appState に書き戻せば復旧できます。不安な場合は、操作前に必ずJSON エクスポートを。
全部やり直したい
⚙️ 設定 → 一番下の全データ消去（確認が2回出ます）。初期モデル3本だけが入った白紙状態に戻ります。消去前のエクスポートを忘れずに。
